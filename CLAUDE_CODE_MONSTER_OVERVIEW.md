# Claude Code Monster — プロダクト概要

> Claude Code のスラッシュコマンド・ルール・エージェント機能だけで構築された、コード不要のテキストベース モンスター育成RPG。

---

## コンセプト

- **GM = Claude Code。** ルールファイル（`.claude/rules/`）がClaudeにGMとしての振る舞いを指示する
- **ダイス = `jot` コマンド。** すべてのランダム要素をシェルコマンドで解決し、結果の改変を禁止
- **セーブデータ = Markdownファイル。** `game-state/` と `.claude/agents/my-monsters/` がデータベース代わり
- **プレイヤー操作 = スラッシュコマンド。** `/explore`, `/battle`, `/gym` 等で進行

## アーキテクチャ

### レイヤー構成

```
┌─────────────────────────────────────────────┐
│  プレイヤー入力（スラッシュコマンド）            │
│  /new-game, /explore, /gym, /status ...      │
├─────────────────────────────────────────────┤
│  GMエンジン（.claude/rules/）                  │
│  game-master.md → battle-system.md            │
│  → state-contract.md                         │
├─────────────────────────────────────────────┤
│  データ参照（.claude/skills/, agents/）         │
│  move-list, type-chart, progression           │
│  monsters-guide, gym-leaders                  │
├─────────────────────────────────────────────┤
│  乱数生成（jot コマンド）                       │
│  d4/d6/d8/d20 をシェルで実行                   │
├─────────────────────────────────────────────┤
│  永続化（game-state/, my-monsters/）           │
│  Markdownファイルの読み書き                     │
└─────────────────────────────────────────────┘
```

### 設計原則

| 原則 | 実現方法 |
|------|---------|
| コードゼロ | 全ロジックをMarkdownのルール記述で表現。プログラムファイルなし |
| 状態の一元管理 | save.md + モンスター個別ファイルが唯一の真実。差分更新禁止、常に全体再生成 |
| 乱数の透明性 | `jot` コマンドの出目をプレイヤーに全公開。GMによる改変を禁止 |
| 責務分離 | ルール（rules/）・データ（skills/agents/）・状態（game-state/）を明確に分離 |
| gitフレンドリー | セーブデータは .gitignore で除外。ルール・データのみバージョン管理 |

## ゲームシステム

### ワールドマップ

```
はじまりの町 → ルート1 → みなとの町（Gym1）→ ルート2 → いわやまの町（Gym2）
                                                         → いかずちのどうくつ → ルート3 → らいうんの町（Gym3）
```

3つのジムバッジ（みなと・いわやま・らいうん）を全取得でゲームクリア。想定プレイ時間: 約2時間。

### モンスター（全10種 = 5進化ライン）

| # | 名前 | タイプ | 役割 | 進化Lv | 進化先 |
|---|------|--------|------|--------|--------|
| 001 | ヒノコ | Fire | 高速アタッカー | 10 | エンカザン |
| 003 | ミズチ | Water | 耐久型 | 10 | リュウカイ |
| 005 | ツボミン | Grass | バランス型 | 10 | モリヌシ |
| 007 | ビリネズ | Elec | 超高速・紙耐久 | 12 | ライコウモリ |
| 009 | ゴロツキ | Normal | HP壁 | 12 | ケモノビト |

- 御三家（001/003/005）はゲーム開始時に1体選択
- 野生（007/009）はルート上で捕獲可能
- 手持ち上限: 3体

### タイプ相性

```
Fire → Grass → Water → Fire（3すくみ）
Grass → Elec → Water（追加の弱点関係）
Normal: 弱点なし・ばつぐんなし
```

### バトルシステム（TRPGスタイル）

1. **行動選択:** 技 / アイテム / 逃走（野生戦のみ）
2. **先攻判定:** d6 + SPD（同値はプレイヤー有利）
3. **命中判定:** d20 + ATK + 技Hit ≥ 12 で命中（nat20=クリティカル、nat1=ファンブル）
4. **ダメージ:** ダイス × タイプ倍率 + ATK − DEF（最低保証1）
5. **ターン終了:** 状態異常処理 → 戦闘不能チェック → state更新

### 技（全20技 = 各タイプ4技）

各技はダメージダイス・命中補正・PP・追加効果を持つ。バトル開始時にPP全回復。

### 状態異常

| 状態 | 効果 |
|------|------|
| やけど | ATK −1 |
| うずまき | SPD −1 |
| マヒ | SPD −2 |
| パラサイト | 毎ターン1d4ダメ＋相手回復 |
| チャージアップ | 次の攻撃 Hit+3, +1d6 |

### 経済

| 項目 | 値段/報酬 |
|------|----------|
| いやしグラス | 50G（HP+10） |
| すごいいやしグラス | 150G（HP+25） |
| トラップボール | 100G（捕獲用） |
| 野生戦勝利 | +10G × 相手Lv |
| トレーナー戦勝利 | +20G × 最高Lv |
| ジム戦勝利 | +50G × 最高Lv |

### 成長

- **XP:** 野生=5+敵Lv、トレーナー=8+最高Lv、ジム=15+最高Lv（参加者全員に同額）
- **レベルアップ:** 必要XP = 次Lv × 4。MaxHP +1/Lv
- **進化:** 指定Lvで自動進化。ステータス・技が進化形に置換

### ジムリーダー

| # | 名前 | タイプ | チーム | 推奨Lv | AI特徴 |
|---|------|--------|--------|--------|--------|
| 1 | カスミ | Water | ミズチLv4 | 3-4 | 素直に攻撃、状況に応じてうずまき |
| 2 | テツ | Normal | ゴロツキLv7 + ケモノビトLv9 | 7-8 | シールド＋チャージの戦略型 |
| 3 | イカヅチ | Elec | ビリネズLv10 + ライコウモリLv12 | 11-12 | マヒ→高火力の戦術型 |

## 要件・制約

### 機能要件

1. プレイヤーはスラッシュコマンドでゲームを操作できる
2. バトルはダイスロール（`jot`）で全判定を行い、結果を改変しない
3. セーブデータはMarkdownファイルで永続化し、セッション間で引き継ぐ
4. モンスターは最大3体まで手持ちに入れられる
5. 3つのジムバッジ取得でゲームクリアとなる
6. オートバトルモードでAIが技選択を自動化できる

### 非機能要件

1. **コードレス:** プログラムファイルを一切使わず、Claude Codeの機能のみで動作する
2. **透明性:** ダイスの出目・ダメージ計算過程をプレイヤーに全公開する
3. **データ整合性:** state-contract.md のスキーマに厳密に従い、差分更新を禁止する
4. **git管理:** ルール・データはバージョン管理し、セーブデータは .gitignore で除外する
5. **拡張性:** モンスター・技・ジムリーダーは個別ファイルで定義され、追加が容易

### 制約

- モンスター総数: 10種（5進化ライン）
- 技総数: 20技（各タイプ4技）
- タイプ数: 5（Fire / Water / Grass / Elec / Normal）
- 最大レベル: 15
- 手持ち上限: 3体
- 動作環境: macOS（`jot` コマンドに依存）
