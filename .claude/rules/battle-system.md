# Battle System — バトルシステム

> **このファイルはゲームの最重要ルールである。GMはバトル中、このファイルの手順に厳密に従うこと。**
> 「雰囲気」や「物語的に面白いから」でダイス結果やダメージ計算を改変してはならない。
> ダイスロールは必ず `jot` コマンドで実行し、結果をそのまま適用すること。

---

## 1. バトルの開始

### バトル種別

| 種別 | 発生条件 | 逃走 | 捕獲 |
|------|---------|------|------|
| 野生戦 | explore コマンドで遭遇 | 可能 | 可能 |
| トレーナー戦 | ストーリー進行で発生 | 不可 | 不可 |
| ジムリーダー戦 | gym コマンドで挑戦 | 不可 | 不可 |

### 開始時の処理

1. `game-state/save.md` を読み込む
2. プレイヤーの先頭モンスターをバトルに出す
3. 相手モンスターの情報を提示する（名前、タイプ、レベル。HPは野生は非公開、ジムリーダーは公開）
4. `game-state/battle-current.md` を作成し、バトル中の一時状態を記録する
5. ターンループに入る

---

## 2. ターン進行

### ターンの流れ

```
┌─────────────────────────────────────────┐
│ 1. プレイヤーの行動選択                    │
│    → 技を選ぶ / アイテムを使う / 逃げる    │
│                                          │
│ 2. 先攻判定（双方が攻撃技を選んだ場合）     │
│    → d6+SPD で比較                        │
│                                          │
│ 3. 先攻側の行動実行                        │
│    → 命中判定 → ダメージ計算 → 適用        │
│                                          │
│ 4. 後攻側の行動実行（戦闘不能でなければ）    │
│    → 命中判定 → ダメージ計算 → 適用        │
│                                          │
│ 5. ターン終了時処理                        │
│    → パラサイトダメージ・回復               │
│    → 戦闘不能チェック                     │
│    → battle-current.md 更新                      │
└─────────────────────────────────────────┘
```

### 行動の選択肢

| 行動 | 効果 | 先攻判定 |
|------|------|---------|
| 技を使う | 選択した技を実行 | 通常通り |
| アイテムを使う | アイテム効果を適用（ターン消費） | アイテムは先攻判定なしで先に実行 |
| 逃げる（野生戦のみ） | d6+SPD ≥ d6+相手SPD で成功 | 失敗したらターン消費、相手が攻撃 |

## 3. 先攻判定

双方が攻撃行動を選んだとき、行動順を決定する。

### 手順

```bash
# プレイヤー側
jot -r 1 1 6    # → d6の結果 + 自分のSPD

# 相手側
jot -r 1 1 6    # → d6の結果 + 相手のSPD
```

**先攻値 = d6 + SPD（状態異常による修正込み）**

- 先攻値が高い方が先に行動する
- **同値の場合:** プレイヤー側が先攻（プレイヤー有利）

### GMの演出

```
⚡ 先攻判定！
   [プレイヤーモンスター]: d6=[X] + SPD[+Y] = [合計]
   [相手モンスター]:       d6=[X] + SPD[+Y] = [合計]
   → [先攻側] が先に動く！
```

## 4. 命中判定

### 基本ルール

**命中ロール = d20 + ATK + 技のHit修正**
**目標値 = 12（固定）**

命中ロール ≥ 12 → **命中**
命中ロール < 12 → **ミス**

```bash
jot -r 1 1 20   # → d20の結果
```

### 特殊ルール

| 出目 | 効果 |
|------|------|
| **d20 = 20**（ナチュラル20） | **クリティカルヒット！** 問答無用で命中＋ダメージダイスを2回振って合算 |
| **d20 = 1**（ナチュラル1） | **ファンブル！** 問答無用でミス（修正値に関わらず） |

### 命中率の目安

ATK+2 のモンスターが各技を使った場合の命中率（d20 ≥ 12 − ATK − Hit）:

| 技Hit | 必要出目 | 命中率 |
|-------|---------|--------|
| +4 | 6以上 | 75% |
| +3 | 7以上 | 70% |
| +2 | 8以上 | 65% |
| +1 | 9以上 | 60% |
| +0 | 10以上 | 55% |
| -1 | 11以上 | 50% |
| -2 | 12以上 | 45% |

### GMの演出

```
🎯 [モンスター名] の [技名]！
   命中判定: d20=[X] + ATK[+Y] + Hit[+Z] = [合計] vs 目標値12
   → [命中！ / ミス…]
```

## 5. ダメージ計算

### 基本式

```
最終ダメージ = Dice × タイプ倍率 + ATK − DEF
              （最低保証: 1ダメージ）
```

### 計算手順（厳密にこの順序で処理すること）

**Step 1: ダイスを振る**

```bash
# 例: 2d6 の場合
jot -r 1 1 6    # 1個目
jot -r 1 1 6    # 2個目
# 合計する
```

**Step 2: タイプ倍率を適用**

`skills/type-chart/SKILL.md` を参照し、攻撃技のタイプ vs 防御側のタイプで倍率を決定。

- ×2（ばつぐん）: Dice合計 × 2
- ×0.5（いまいち）: Dice合計 ÷ 2（端数切り捨て、最低1）
- ×1（等倍）: そのまま

**Step 3: ATK を加算**

タイプ倍率適用後の値に、攻撃側の ATK を足す。

**Step 4: DEF を減算**

防御側の DEF を引く。

**Step 5: 最低保証**

結果が 1 未満なら 1 にする。

### 計算例

```
エンカザン(ATK+3) の バーストフレア(Fire, 3d6, Hit-1) vs リュウカイ(Water, DEF+4)

Step 1: 3d6 = [4] + [2] + [5] = 11
Step 2: Fire vs Water = ×0.5 → 11 ÷ 2 = 5（端数切り捨て）
Step 3: 5 + ATK(3) = 8
Step 4: 8 − DEF(4) = 4
Step 5: 4 ≥ 1 → 最終ダメージ: 4

エンカザン(ATK+3) の バーストフレア(Fire, 3d6, Hit-1) vs ツボミン(Grass, DEF+1)

Step 1: 3d6 = [4] + [2] + [5] = 11
Step 2: Fire vs Grass = ×2 → 11 × 2 = 22
Step 3: 22 + ATK(3) = 25
Step 4: 25 − DEF(1) = 24
Step 5: 24 ≥ 1 → 最終ダメージ: 24  ← ばつぐんは本当に痛い！
```

### クリティカルヒット時

- ダメージダイスを **2回振って合算** する（例: 2d6 → 4d6 として振る）
- それ以降の計算手順（タイプ倍率→ATK→DEF）は通常通り

### GMの演出

```
💥 [技名] が命中！
   ダメージ: [Dice結果] [×2 ばつぐん！ / ×0.5 いまいち…]
   + ATK(+[値]) − DEF(+[値]) = [最終ダメージ] ダメージ！
   [相手モンスター] の HP: [残HP] / [MaxHP]
```

## 6. 状態異常の処理

### 状態異常一覧

| 状態 | 効果 | 発生源 | 持続 |
|------|------|--------|------|
| やけど | ATK −1 | ヒートブレス | バトル終了まで |
| うずまき | SPD −1 | うずまき | バトル終了まで |
| マヒ | SPD −2 | しびれウェーブ, サンダーラッシュ(1/3) | バトル終了まで |
| パラサイト | ターン終了時 1d4 ダメ＋相手同量回復 | パラサイト | バトル終了まで |
| チャージアップ | 次の攻撃 Hit+3, ダイス+1d6 | チャージアップ | 次攻撃時消費 |

### 処理ルール

- **重複なし:** 同じ状態異常を重ねがけしても効果は更新されない
- **複数種の共存:** 異なる状態異常は同時にかかる（やけど＋マヒ など）
- **ステータス修正は元の値から計算:** やけど(ATK-1)とマヒ(SPD-2)が同時なら、元のATKから-1、元のSPDから-2
- **チャージアップは攻撃使用時に消費:** 変化技（シールド等）では消費しない

### サンダーラッシュのマヒ判定

```bash
jot -r 1 1 6    # 1〜2 でマヒ発生、3〜6 で何も起きない
```

### ターン終了時の処理順

1. パラサイトダメージ（`jot -r 1 1 4` で 1d4）→ 同量を植え付けた側のHP回復
2. 戦闘不能チェック（HP ≤ 0 のモンスターがいるか）

## 7. 戦闘不能と後続

### 戦闘不能

HP が 0 以下になったモンスターは **戦闘不能** になる。HPは 0 で止める（マイナスにしない）。

### 後続モンスターの選出

- **プレイヤー側:** 戦闘不能になったら、手持ちから次のモンスターを選ぶ（プレイヤーが選択）
- **相手側（トレーナー/ジム戦）:** あらかじめ決められた順序で出す
- **交代のタイミング:** 戦闘不能になった直後。交代はターンを消費しない（次のターンから行動可能）
- **任意交代は不可:** バトル中に自分から手持ちを入れ替えることはできない

### 勝敗条件

| 条件 | 結果 |
|------|------|
| 相手の全モンスターが戦闘不能 | **プレイヤー勝利** |
| プレイヤーの全モンスターが戦闘不能 | **プレイヤー敗北** |

### 敗北時の処理

- 手持ち全員のHPを MaxHP の半分まで回復（端数切り上げ）
- 所持金の 1/4 を失う（端数切り捨て）
- 最後に訪れた回復ポイント（町）に戻る
- **ジム戦での敗北:** 再挑戦可能（相手のチームは同じ）

## 8. 捕獲システム

### 捕獲条件

- **野生戦のみ** 捕獲可能（トレーナー戦・ジム戦は不可）
- 捕獲はアイテム「トラップボール」を使用する（ターン消費）
- 手持ちが 3体上限。3体いる場合は捕獲を試みられない

### 捕獲判定

```
捕獲値 = d20 + 捕獲ボーナス
目標値 = 8 + 相手レベル − HPボーナス − 状態ボーナス
```

**HPボーナス（相手の残HP割合）:**

| 残HP | ボーナス |
|------|---------|
| 75%以上 | +0 |
| 50〜74% | +2 |
| 25〜49% | +4 |
| 25%未満 | +6 |

**状態ボーナス:**

| 状態異常 | ボーナス |
|---------|---------|
| なし | +0 |
| やけど / うずまき | +1 |
| マヒ | +2 |

**捕獲値 ≥ 目標値 → 捕獲成功**

```bash
jot -r 1 1 20   # 捕獲ロール
```

### GMの演出

```
⚪ トラップボールを投げた！
   捕獲ロール: d20=[X] + ボーナス[+Y] = [合計]
   目標値: 8 + Lv[相手Lv] − HP補正[値] − 状態補正[値] = [目標値]
   → [捕獲成功！ / …惜しい！ボールから出てしまった！]
```

## 9. 逃走

### 逃走条件

- 野生戦のみ可能

### 逃走判定

```
プレイヤー: d6 + 自分のSPD
相手:       d6 + 相手のSPD
```

プレイヤー側 ≥ 相手側 → **逃走成功**（バトル終了、XPなし）
プレイヤー側 < 相手側 → **逃走失敗**（ターン消費、相手が攻撃してくる）

```bash
jot -r 1 1 6   # プレイヤー側
jot -r 1 1 6   # 相手側
```

## 10. NPCの行動AI

### 野生モンスター

- **ランダム:** 使用可能な技からランダムに選択する

```bash
# 技が3つある場合
jot -r 1 1 3
```

### トレーナー

- **セミランダム:** タイプ相性で有利な技を優先。有利な技がなければランダム

### ジムリーダー

- **戦術的:** `.claude/agents/gym-leaders/` に定義された AI 性格に従う（詳細は .claude/agents/gym-leaders/ を参照）

### プレイヤーオートAI（auto_battle = true のとき）

オートモードが ON のとき、プレイヤーのわざせんたくを以下のルールでじどうけっていする。

**ゆうせんじゅんい（うえからじゅんにはんだん）:**

1. **かいふくはんだん:** じぶんの HP が MaxHP の 1/4 いか、かつ「いやしグラス」か「すごいいやしグラス」をもっている → アイテムをつかう（すごいいやしグラスゆうせん）
2. **ばつぐんわざがある:** あいてタイプに ×2 のわざをもっていて PP がのこっている → そのわざをつかう（ふくすうあればダメージダイスのおおきいほう）
3. **とうばいわざでいちばんつよいもの:** あいてタイプに ×1 のわざのなかで、ダメージきたいちがいちばんたかいわざをつかう
4. **PP がのこっているわざ:** うえのどれにもあてはまらなければ、PP がのこっているわざからランダム

**ダメージきたいちのけいさん:**
```
きたいち = (ダイスのへいきんち × タイプばいりつ) + ATK − あいてDEF
```
- 1d6 のへいきん = 3.5
- 2d6 のへいきん = 7
- 3d6 のへいきん = 10.5
- 1d4 のへいきん = 2.5
- 1d8 のへいきん = 4.5

**オートAIがやらないこと:**
- ほかく（トラップボールをなげる）→ プレイヤーにかくにんする
- にげる → プレイヤーにかくにんする
- わざわすれ・しんかじのわざせんたく → プレイヤーにかくにんする
- チャージアップ → サンバーストのコンボ（ためわざのはんだんはしない。たんぱつわざのみえらぶ）

**GMの演出（オートモードじ）:**
```
🤖 [モンスターめい] は [わざめい] をえらんだ！（AUTO）
```
→ いつもどおりの はんていにすすむ

## 11. バトル終了処理

### 勝利時の処理手順

1. XP を計算する（`skills/progression/SKILL.md` 参照）
2. 参加モンスター全員に XP を付与する
3. レベルアップ判定を行う
4. レベルアップした場合:
   a. MaxHP +1、現在HP +1
   b. 技習得チェック（`skills/progression/SKILL.md` の習得スケジュール参照）
   c. 進化チェック（進化レベルに達したか）
5. `game-state/battle-current.md` を `game-state/battles/{つうばん}-{けっか}.md` にリネームしてほぞん
6. `game-state/save.md` を更新する

### GMのバトル終了演出

```
🏆 [相手モンスター名] を倒した！
   獲得XP: [値]

   📊 [モンスター名1]: XP [現在値]/[次Lvまで]
   [レベルアップした場合]
   🎉 [モンスター名] はレベル [N] になった！
      MaxHP: [旧] → [新]
      [技習得があれば表示]
      [進化があれば skills/progression/SKILL.md の進化演出を実行]
```

## 12. ダイスロール実行規約

### 絶対ルール

1. **すべてのダイスは `jot` コマンドで振ること。** 頭の中で「適当な数字」を決めてはならない。
2. **1ターンに必要な全ダイスをまとめて振ってよい** が、各ロールの対応を明示すること。
3. **結果は改変不可。** ナチュラル1が出ても、3d6が全部1でも、そのまま適用する。
4. **ダイスロールのログを battle-current.md に記録すること。**

### jot コマンドリファレンス

```bash
# d6 を1個
jot -r 1 1 6

# d20 を1個
jot -r 1 1 20

# d4 を1個
jot -r 1 1 4

# d8 を1個
jot -r 1 1 8

# 複数ダイスをまとめて（例: 先攻d6×2, 命中d20×2, ダメージ2d6+1d6）
# → 1回のコマンドで全部振るのではなく、用途ごとに分けて振ること
```
