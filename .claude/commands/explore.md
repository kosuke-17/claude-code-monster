# Command: /explore

> フィールドを探索し、野生モンスターとの遭遇や次のエリアへの移動を行う。

## トリガー

プレイヤーが `/explore`、「探索する」、「進む」、「歩く」、「次に進む」等と発言したとき。

## 処理手順

1. **save.md を読み込む**（現在地を確認）+ **`.claude/agents/my-monsters/` を読み込む**（手持ち状態を確認）

2. **現在地が町の場合**
   ```
   📍 [町名] にいます。
   どこへ向かいますか？
   [行ける場所のリストを game-master.md のワールドマップから提示]
   ```
   - プレイヤーが行き先を選んだら Location を更新

3. **現在地がルート/洞窟の場合 → エンカウント判定**
   ```bash
   jot -r 1 1 6   # 1〜5: エンカウント、6: 何もなし
   ```

   **エンカウントなし（6）:**
   ```
   🚶 [エリア名] を進んでいく…
   [1〜2行の情景描写をランダムに]
   もう少し進みますか？ それとも引き返しますか？
   ```

   **エンカウントあり（1〜5）:**
   - `game-master.md` セクション4 に従いモンスター種・レベルを決定
   - 自動的に `/battle` の野生戦フローに移行
   ```
   🚶 [エリア名] を進んでいると…ガサガサッ！
   野生の [モンスター名] (Lv[N]) が飛び出してきた！
   → バトル開始！
   ```

4. **エリア踏破判定**
   - 同一エリアで **3回の探索**（エンカウント有無問わず）を行うと、次のエリアへの道が開く
   - 踏破時:
   ```
   🛤️ [エリア名] を抜けた！ [次のエリア/町名] が見えてきた。
   [次のエリア/町名] に到着した！
   ```
   - Location を更新

## 情景描写のバリエーション（GMが適宜選ぶ）

**ルート1（草原）:**
- 「穏やかな風が草むらを揺らしている」
- 「遠くで鳥の鳴き声が聞こえる」
- 「木漏れ日が差し込む小道を歩く」

**ルート2（丘陵）:**
- 「坂道が続く。遠くに山が見える」
- 「風が少し冷たくなってきた」
- 「岩場の間を縫うように進む」

**いかずちのどうくつ:**
- 「洞窟の奥から微かに光が見える」
- 「壁に触れると、ビリッと静電気が走った」
- 「水滴の音が反響している」

**ルート3（山道）:**
- 「険しい山道。息が上がる」
- 「雷雲が遠くに見える」
- 「強い風が吹きつける」

## 注意

- 先頭モンスターのHPが 0 の場合、「先に回復した方がいい」と警告する（探索自体は許可）
- 手持ち全員が戦闘不能の場合は探索できない → 町に強制帰還
